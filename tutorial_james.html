James tutorial

James is the name of a humanoid robot project developed by University of Genoa and Italian Institute of Technology (IIT), intended to operate in the context of object manipulation. The dataset used here is a collection composed of 8 inputs and 6 outputs: the eight inputs consist of two vectors of four elements each, one related to the velocity and the other to the angle of the shoulder and elbow joints. The six outputs are the corresponding values of internal forces and torques in a Cartesian space. These data are meant to detect potentially unsafe situations, e.g., contact with obstacles or humans.

Network Overview

The picture below shows the formal representation of the linear Neural Network for this dataset. The architecture consists of 3 hidden layers with ReLU activation functions and a decreasing number of neurons, namely 64, 32 and 16.
This network takes as input a 1D vector of 8 elements and outputs a 1D vector of 6 elements.


Network Construction


In NeVer2 you can select the necessary blocks from the left toolbar; the requested nodes will appear on the canvas with default parameters. You can drag and drop them to any position in the canvas. To edit their parameters you can just double-click or right-click -> "Edit", while to delete them you can press DEL or right-click -> "Delete".


The first layer is defined by a Fully Connected block, with the parameters defined as in the example; the input shape is 8 and the out_features parameter is set to 64, i.e., the dimension of the first hidden layer.


After this you can add a ReLU block as an activation function. You can connect the two blocks activating the drawing mode with the "Draw Line" button in the top-left corner or by pressing the CTRL+D shortcut and clicking first on the starting block and then on the next block.


For the next layer you can add a new Fully Connected block with the out_features parameter set to 32, and connecting it to the first ReLU node. Now the in_features parameter of the second Fully Connected node is updated with the ReLU output value (64).


The sequence of Fully Connected and ReLU nodes is repeated with the structure presented in the beginning. At the end of the last ReLU node you can finally add the Fully Connected output layer, and set the out_features parameter to 6.

Network Training


Once the network is completed, you can train it. In the menu bar select "Learn..." -> "Train": the training window appears, where you can select the dataset and the learning parameters. We made available the most common datasets directly in the application, but in order to load a custom dataset some additional steps are required. By selecting Custom Data Source... in the Dataset box you can load the corresponding file, and fill the dataset parameters dialog.

In particular, the Target index is the separator between input and output data, i.e., the number of inputs. The data type is controlled by a combo box and the delimiter character is set by default to a comma.

The learning parameters can be set as follows:

Optimizer: at the moment only "Adam" (adaptive moment estimation) is supported, being the most popular and effective gradient based optimization algorithm. Once selected, you can edit all its related parameters.

Learning Rate Scheduler: like the Optimizer, only "ReduceLROnPlateau" is supported. It adjusts the learning rate only when a plateau in model performance is detected, e.g. no change for a given number of training epochs.

Loss Function: you can choose between "Cross Entropy" and "MSE Loss", depending on the network structure.

Precision Metric: you can choose between "Inaccuracy" and "MSE loss".

Epochs: it defines the steps that the learning algorithm will perform through the training process.

Validation Percentage: (a number between 0 and 1) indicates the percentage of the dataset to use as the validation set.

Training batch size and Validation batch size: it defines the dimension of the training batches.

Cuda: it allows to take advantage of NVidia GPU architecture for the computation.

Train patience (optional): the number of epochs in which the loss may not decrease before the training procedure is interrupted with early stopping.

Checkpoints root (optional): where to store the checkpoints of the training strategy, by default the working directory of NeVer2.

Verbosity level (optional): how many batches between training log prints (default 1).


Once the parameters are set, clicking "Train Network" runs the training. Logs are printed in a box below the buttons.


The Best Loss Score label indicates the training error percentage; in this case, with a 10 epoch long training the best loss is 0.00048, corresponding to a 0.048% error rate. Looking at the following diagram it is clear that increasing the training epochs improves slightly the error rate.
